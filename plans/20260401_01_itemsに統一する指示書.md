# Random Picker 修正指示書
## 目的
文字列配列を表す識別子が `values` と `items` で混在しているため、**文字列配列を表す名前を `items` に統一**する。

今回の修正は **命名の統一のみ** が目的である。
**挙動変更は禁止**。
**ロジック変更は禁止**。
**型の意味変更は禁止**。
**関数の追加・削除は禁止**。
**ファイル追加・削除は禁止**。

---

## 最重要ルール
今回変更してよいのは、**「文字列配列そのもの」を表している識別子名だけ**です。

### 変更してよいもの
- 関数引数名
- ローカル変数名
- コールバック引数名
- 型エイリアス内の引数名
- テスト名の文字列
- テスト内のモック関数の引数名

### 変更してはいけないもの
- 関数名
- export 名
- ファイル名
- import/export 構造
- 処理順
- 条件分岐
- 戻り値
- 例外
- 画面文言
- DOM 要素を表す `item`
- 単一値を表す `value`
- URL 配列を表す `urls`
- 入力行/出力行を表す `inputLines`, `outputLines`
- `excludedItems` など、すでに意味が明確な別名

特に重要:
- `value` は「単一の文字列」や「単一の値」を表すことがあるので、**機械的に全部 `item` に変えないこと**
- `item` は DOM 要素を表している箇所もあるので、**DOM の `item` には触らないこと**
- 今回の統一対象は **「文字列配列」だけ**

---

## 命名方針
以下に統一すること。

- 文字列配列全体: `items`
- 文字列配列の各要素: `item`

今回 **`values` は使わない**。

## ファイルごとの具体的な修正内容

### 1. `apps/random-picker/src/type.ts`
`StringArrayProcessor` の引数名を `values` から `items` に変える。

#### 修正前
```ts
export type StringArrayProcessor = (values: string[]) => string[];
````

#### 修正後

```ts
export type StringArrayProcessor = (items: string[]) => string[];
```

注意:

* 型自体の意味は変えない
* `string[]` のまま
* 戻り値も `string[]` のまま

---

### 2. `apps/random-picker/src/pipeline.ts`

以下を修正する。

#### 2-1. `applyStringArrayProcessors`

引数名 `values` を `items` に変更する。

また、`reduce` のアキュムレータ名 `currentValues` も `currentItems` に変更する。

#### 修正前

```ts
export function applyStringArrayProcessors(
  values: string[],
  processors: StringArrayProcessor[],
): string[] {
  return processors.reduce(
    (currentValues, processor) => processor(currentValues),
    values,
  );
}
```

#### 修正後

```ts
export function applyStringArrayProcessors(
  items: string[],
  processors: StringArrayProcessor[],
): string[] {
  return processors.reduce(
    (currentItems, processor) => processor(currentItems),
    items,
  );
}
```

重要:

* `reduce` の処理内容は変えない
* ただの名前変更に留める

#### 2-2. `joinByNewline`

引数名 `values` を `items` に変更する。

#### 修正前

```ts
export function joinByNewline(values: string[]): string {
  return values.join("\n");
}
```

#### 修正後

```ts
export function joinByNewline(items: string[]): string {
  return items.join("\n");
}
```

重要:

* `.join("\n")` の処理は変えない

#### 2-3. それ以外

`executePipeline` の `currentItems` は既に適切なので変更不要。
`splitByNewline` の `sourceText` も適切なので変更不要。

---

### 3. `apps/random-picker/src/processor.ts`

以下を修正する。

#### 3-1. `filterEmptyStrings`

引数名 `values` を `items` に変更する。
コールバック引数名 `value` も `item` に変更する。

#### 修正前

```ts
export function filterEmptyStrings(values: string[]): string[] {
  return values.filter((value) => value !== "");
}
```

#### 修正後

```ts
export function filterEmptyStrings(items: string[]): string[] {
  return items.filter((item) => item !== "");
}
```

重要:

* フィルタ条件は変えない
* 空文字列除外の仕様はそのまま

#### 3-2. `trimStrings`

引数名 `values` を `items` に変更する。
コールバック引数名 `value` も `item` に変更する。

#### 修正前

```ts
export function trimStrings(values: string[]): string[] {
  return values.map((value) => value.trim());
}
```

#### 修正後

```ts
export function trimStrings(items: string[]): string[] {
  return items.map((item) => item.trim());
}
```

重要:

* `trim()` の処理は変えない

#### 3-3. `pickRandomItems`

既に `items` なので変更不要。

#### 3-4. `removeExcludedItems`

既に `items` / `excludedItems` で意味が明確なので変更不要。

### 4. `apps/random-picker/tests/unit/pipeline.test.ts`

`applyStringArrayProcessors` のテスト内に `values` 表記が残っているので、**文字列配列を表すものだけ** `items` に変更する。

#### 修正対象

* `vi.fn<(values: string[]) => string[]>()` → `vi.fn<(items: string[]) => string[]>()`
* `.mockImplementation((values) => ...)` → `.mockImplementation((items) => ...)`
* テスト名 `"values.length=0 / ..."` → `"items.length=0 / ..."`

#### 修正イメージ

```ts
const trimItems = vi
  .fn<(items: string[]) => string[]>()
  .mockImplementation((items) => items.map((item) => item.trim()));

const addSuffix = vi
  .fn<(items: string[]) => string[]>()
  .mockImplementation((items) => items.map((item) => `${item}!`));
```

重要:

* モックの処理内容は変えない
* `trimValues` という変数名も、必要なら `trimItems` に変えてよい
* ただし `addSuffix` のように `values` を含まない名前は無理に変えなくてよい
* `expect` の期待値は一切変えない

#### 境界系テスト名

```ts
test("values.length=0 / 配列に処理関数が順に適用される", ...)
```

を

```ts
test("items.length=0 / 配列に処理関数が順に適用される", ...)
```

に変える。

---

### 5. `apps/random-picker/tests/unit/processor.test.ts`

`values` 表記が残っている箇所だけ修正する。

#### 修正対象

* テスト名 `"values.length=0"` → `"items.length=0"`

必要に応じて、今回修正した関数に対応するローカルな型引数やコールバック引数で `values` が残っていれば `items` に揃える。
ただし、このファイルでは多くが既に `items` なので、**不要な変更はしないこと**。

## 修正しないことの具体例

以下は修正禁止。

### 修正禁止例1

```ts
export async function copyTextToClipboard(value: string): Promise<void>
```

この `value` は配列ではないので変更禁止。

### 修正禁止例2

```ts
function handlePipelineStepListDragStart(...) {
  const item = target.closest(".pipeline-step-item") as HTMLElement;
}
```

この `item` は DOM 要素なので変更禁止。

### 修正禁止例3

```ts
const urls = applyStringArrayProcessors(inputLines, [...]);
```

この `urls` は URL 配列を表していて意味が明確なので変更禁止。

### 修正禁止例4

```ts
const inputLines = splitByNewline(inputText);
const outputLines = splitByNewline(outputText);
```

これは「行」であり、今回の統一対象ではないので変更禁止。

### 修正禁止例5

```ts
removeExcludedItems(items, excludedItems)
```

`excludedItems` は除外対象の意味があるので変更禁止。

`values` で全置換すると壊れる可能性がある。
**必ず1件ずつ意味を見て修正すること。**

特に注意する検索語:

* `values: string[]`
* `(values)`
* `currentValues`
* `"values.length=0"`
